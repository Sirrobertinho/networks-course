### Задание А (3 балла)
Создайте простой REST сервис, в котором используются HTTP операции GET, POST, PUT и DELETE.
Предположим, что это сервис для будущего интернет-магазина, который пока что умеет 
работать только со списком продуктов. У каждого продукта есть поля: `id` (уникальный идентификатор),
`name` и `description`. 

Таким образом, json-схема продукта (обозначим её `<product-json>`):

```json
{
  "id": 0,
  "name": "string",
  "description": "string"
}
```

Данные продукта от клиента к серверу должны слаться в теле запроса в виде json-а, **не** в параметрах запроса.

Ваш сервис должен поддерживать следующие операции:
1. Добавить новый продукт. При этом его `id` должен сгенерироваться автоматически
   - `POST /product`
   - Схема запроса:
     ```json
     {
       "name": "string",
       "description": "string"
     }
     ```
   - Схема ответа: `<product-json>` (созданный продукт)
2. Получить продукт по его id
   - `GET /product/{product_id}`
   - Схема ответа: `<product-json>`
3. Обновить существующий продукт (обновляются только те поля продукта, которые были переданы в теле запроса)
   - `PUT /product/{product_id}`
   - Схема запроса: `<product-json>` (некоторые поля могут быть опущены)
   - Схема ответа: `<product-json>` (обновлённый продукт)
4. Удалить продукт по его id
   - `DELETE /product/{product_id}`
   - Схема ответа: `<product-json>` (удалённый продукт)
5. Получить список всех продуктов 
   - `GET /products`  
   - Схема ответа:
     ```
     [ 
       <product-json-1>,
       <product-json-2>, 
       ... 
     ]
     ```

Предусмотрите возвращение ошибок (например, если запрашиваемого продукта не существует).

Вы можете положить код сервиса в отдельную директорию рядом с этим документом.

## Задачи

### Задача 1 (2 балла)
Общая (сквозная) задержка прохождения для одного пакета от источника к приемнику по пути,
состоящему из $N$ соединений, имеющих каждый скорость $R$ (то есть между источником и
приемником $N - 1$ маршрутизатор), равна $d_{\text{сквозная}} = N \dfrac{L}{R}$
Обобщите данную формулу для случая пересылки количества пакетов, равного $P$.

#### Решение
Заметим, что первый пакет придёт через $N \cdot \dfrac{L}{R}$ времени. Передача пакетов непрерывная, поэтому через время $\dfrac{L}{R}$ придёт второй пакет. Каждый следующий будет приходить через $\dfrac{L}{R}$ времени после предыдущего(так как следующий пакет отправляется по соединению сразу после отправки предыдущего). Таких пакетов $P - 1$. Значит общее время задержки $= N \cdot \dfrac{L}{R} + (P-1) \cdot \dfrac{L}{R} = (N+P-1) \cdot \dfrac{L}{R}$  
Ответ: $(N+P-1) \cdot \dfrac{L}{R}$.

### Задача 2 (2 балла)
Допустим, мы хотим коммутацией пакетов отправить файл с хоста A на хост Б. Между хостами установлены три
последовательных канала соединения со следующими скоростями передачи данных:
$R_1 = 200$ Кбит/с, $R_2 = 3$ Мбит/с и $R_3 = 2$ Мбит/с.
Сколько времени приблизительно займет передача на хост Б файла размером $5$ мегабайт?
Как это время зависит от размера пакета?

#### Решение
Пусть размер пакета - $L$, пакетов - $n$, тогда общий размер файла $X = n \cdot L$. Тогда первый пакет придёт через время: $\dfrac{L}{R_{1}} + \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}}$. Заметим, что следующий пакет придёт через время $\dfrac{L}{min(R_{1}, R_{2}, R_{3})}$, так как пакеты идут непрерывно, а максимальная задержка между пакетами как раз будет при прохождении через канал с минимальной скоростью передачи данных. Тогда раз $R_{1}$ - минимально, то общее время будет: $(\sum_{i = 1}^{n} \dfrac{L}{R_{1}}) + \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}} = n \cdot \dfrac{L}{R_{1}} + \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}} = \dfrac{X}{R_{1}} + \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}}$. Подставляя числа из условия: $X =  5\ MB, R_{1} = 200\ Kb/s$, получаем ($5 MB = 40 Mb = 40000 Kb$): $\dfrac{40000}{200} + \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}} =$ 200 секунд $+ \dfrac{L}{R_{2}} + \dfrac{L}{R_{3}}$.  
Ответ: приблизительно 200 секунд.  
От размера пакета время зависит так: 200 секунд будет точно + оставшаяся часть зависит линейно от размера пакета: $\dfrac{L}{R_{2}} + \dfrac{L}{R_{3}}$. При малых размерах пакета этим временем можно пренебречь, а при размерах пакета сопоставивыми со скоростью передачи данных, нужно это учитывать.

### Задача 3 (2 балла)
Предположим, что пользователи делят канал с пропускной способностью $2$ Мбит/с. Каждому
пользователю для передачи данных необходима скорость $100$ Кбит/с, но передает он данные
только в течение $20$ процентов времени использования канала. Предположим, что в сети всего $60$
пользователей. А также предполагается, что используется сеть с коммутацией пакетов. Найдите
вероятность одновременной передачи данных $12$ или более пользователями.

#### Решение
Можно считать, что количество пользователей $X$ описывается биномиальным распределением: $X \sim Binom(n, p)$, где $n$ = 60, $p$ = 0.2, так как у нас 60 людей и занимают они 20% времени. Тогда если мы хотим посчитать $P(X \geq 12) = 1 - P(X \leq 11) = \sum_{i = 0}^{11} P(X = i) = \sum_{i = 0}^{11} \dbinom{60}{i} \cdot p^{i} (1 - p)^{60 - i}$. Я по-честному посчитал, запрограмировав вычисление, и получил: 0.5513825262506507.  
Ответ: 0.5513825262506507.

### Задача 4 (2 балла)
Пусть файл размером $X$ бит отправляется с хоста А на хост Б, между которыми три линии связи и
два коммутатора. Хост А разбивает файл на сегменты по $S$ бит каждый и добавляет к ним
заголовки размером $80$ бит, формируя тем самым пакеты длиной $L = 80 + S$ бит. Скорость
передачи данных по каждой линии составляет $R$ бит/с. Загрузка линий мала, и очередей пакетов
нет. При каком значении $S$ задержка передачи файла между хостами А и Б будет минимальной?
Задержкой распространения сигнала пренебречь.

#### Решение
Пусть $n$ - количество пакетов, тогда $n = \dfrac{X}{S}$. Заметим, что первый пакет придёт через время $\dfrac{L}{R} + \dfrac{L}{R} + \dfrac{L}{R} = \dfrac{3L}{R}$. Так как скорость передачи данных на каждой линии совпадает и передача пакетов непрерывная, то следующий пакет будет приходить через $\dfrac{L}{R}$ после предыдущего. Тогда общая задержка передачи: $T(S) = \dfrac{3L}{R} + \sum_{i = 2}^{n} \dfrac{L}{R} = (n + 2) \cdot \dfrac{L}{R} = (\dfrac{X}{S} + 2) \cdot (\dfrac{S + 80}{R})$. Берёи производную: $T'(S) = \dfrac{1}{R} \cdot (\dfrac{-80X}{S^{2}} + 2)$. Довольно очевидно, что производная по $S$ возрастает(при $S > 0$), значит так как при $S = 0+: T'(S) < 0$ , при $S = \sqrt{40X}: T'(S) = 0$, при $S > \sqrt{40X}: T'(S) > 0$, то минимум функции $T(S)$ будет в точке: $S = \sqrt{40X}$.  
Ответ: $S = \sqrt{40X}$.

### Задание 5 (2 балла)
Рассмотрим задержку ожидания в буфере маршрутизатора. Обозначим через $I$ интенсивность
трафика, то есть $I = \dfrac{L a}{R}$.
Предположим, что для $I < 1$ задержка ожидания вычисляется как $\dfrac{I \cdot L}{R (1 – I)}$. 
1. Напишите формулу для общей задержки, то есть суммы задержек ожидания и передачи.
2. Опишите зависимость величины общей задержки от значения $\dfrac{L}{R}$.

#### Решение
Общая задержка состоит из задержки ожидания + задержка передачи. Задержка передачи: $\dfrac{L}{R}$, по условию задержка ожидания: $\dfrac{I \cdot L}{R (1 – I)}$, тогда общая задержка: $\dfrac{L}{R} + \dfrac{I \cdot L}{R (1 – I)} = \dfrac{L}{R} \cdot (1 + \dfrac{I}{1 - I}) = \dfrac{L}{R} \cdot \dfrac{1}{1 - I} = \dfrac{L}{R} \cdot \frac{1}{1 - \dfrac{La}{R}}$.  
Ответ: 1. $\dfrac{L}{R} \cdot \frac{1}{1 - \dfrac{La}{R}}$.  
Обозначим $x = \dfrac{L}{R}$. Тогда общая задержка: $\dfrac{x}{1-xa}$. Эта формула зависит дробно-линейно от $x$. Можно упомянуть, что при $I \sim 0$ зависимость почти линейная($I = xa, I \sim 0, 1-xa \sim 1, \dfrac{x}{1-xa} \sim x$), а при $I \sim 1$ задержка бесконечно большая.  
Ответ: 2. Дробно-линейная зависимость.
